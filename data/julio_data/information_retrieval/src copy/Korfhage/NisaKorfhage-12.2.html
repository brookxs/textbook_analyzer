12.2 Processing Scanned Documents

When a document is scanned into a computer, a bit map, or pictorial image of the document is created. Then conversion to a text file format is required, using the OCR. Smudged, misalignment, skewed printing and incomplete printing may result in very poor OCR performance.

Several techniques are employed in OCR software to improve recognition capabilities:

  • To make use of context to resolve ambiguities in recognition.
  • Graphical alterations of characters such as assign missing bits or adding an extra border to a character.
  • Segmentation of the document image based on the fact that text image usually have several components that may be a title block, sentence or paragraphs.

Basic output from the OCR processing is an ASCII text file but there is also an increasing interest in incorporating SGML or HTML as the resulting file.
